Comprendre Strapi

Le CMS Headless Open-Source basé sur Node.js

1. Qu'est-ce que Strapi ?

Strapi est un CMS (Content Management System) dit "Headless". Contrairement à un CMS traditionnel comme WordPress, Strapi ne gère pas l'affichage final (le front-end). Il se concentre uniquement sur la gestion du contenu (back-office) et expose ce contenu via des API (REST et GraphQL) pour qu'il puisse être consommé par n'importe quel client (site web React/Vue/Angular, application mobile Flutter/React Native, objets connectés, etc.).

Pourquoi l'utiliser ?

  • Gains de temps : Crée des API performantes en quelques clics sans écrire de code de routage complexe.
  • Agnostique au Front-end : Utilise ta technologie préférée (Next.js, Flutter, etc.) pour créer l'interface utilisateur.
  • Hautement personnalisable : Basé sur Node.js, son code source est accessible et modifiable.

2. Technologies Utilisées par Strapi

Strapi est construit sur une stack JavaScript/TypeScript moderne :

  • Back-end (Serveur) : Node.js avec le framework Koa.js (qui gère les requêtes HTTP).
  • Front-end (Panel d'administration) : Construit avec React.js.
  • Base de données : Strapi utilise un ORM puissant pour se connecter à diverses bases de données. Il supporte : PostgreSQL, MySQL, MariaDB et SQLite.

3. Les Concepts Fondamentaux (Le Cœur de Strapi)

Pour maîtriser Strapi, il faut comprendre comment il structure les données :

A. Content-Types Builder (Le Constructeur de Modèles)

C'est l'interface où tu définis la structure de ta base de données sans écrire de SQL. Tu y crées des "modèles" de contenu.

B. Collection Types (Types de Collection)

Utilisés pour les données répétitives. Exemple : Des Articles, des Produits, des Utilisateurs. Si tu as besoin d'une liste de contenus, c'est un Collection Type.

C. Single Types (Types Uniques)

Utilisés pour les données uniques qui n'existent qu'en un seul exemplaire. Exemple : La page "À propos", le "Menu de navigation principal", les "Paramètres SEO globaux".

D. Components (Composants)

Ce sont des structures de données réutilisables. Exemple : Un composant "Bouton" (contenant un lien et un texte) ou un composant "Section Hero" (titre, image intro). Tu peux inclure ces composants dans tes Collection ou Single Types.

E. Dynamic Zones (Zones Dynamiques)

C'est la fonctionnalité la plus puissante pour les créateurs de contenu. Une "Dynamic Zone" permet à l'administrateur d'insérer différents composants dans l'ordre qu'il souhaite (par exemple, construire un article avec un paragraphe, puis une image, puis un carrousel, etc.).

4. Fonctionnalités Clés

A. API REST et GraphQL

Dès que tu crées un Collection Type (ex: "Article"), Strapi génère automatiquement les routes d'API :

  • GET /api/articles (Lister)
  • GET /api/articles/:id (Récupérer un)
  • POST /api/articles (Créer)
  • PUT /api/articles/:id (Modifier)
  • DELETE /api/articles/:id (Supprimer)

B. Roles & Permissions (RBAC)

Strapi possède un système très fin de gestion des droits :

  • Public : Ce que les utilisateurs non connectés peuvent voir (ex: lire les articles).
  • Authenticated : Ce que les utilisateurs connectés peuvent faire (ex: laisser un commentaire).
  • Admin Roles : Contrôle précis de ce que les éditeurs, auteurs et super-admins peuvent faire dans le panneau d'administration.

C. Media Library (Médiathèque)

Interface pour uploader, modifier (recadrer) et optimiser les images, vidéos et fichiers. Il supporte les "Providers" externes (ex: Cloudinary, AWS S3) pour stocker les fichiers dans le cloud plutôt que sur le serveur.

D. Webhooks

Strapi peut avertir le monde extérieur d'un changement. Exemple : À chaque fois qu'un article est publié, Strapi envoie une requête (Webhook) à Vercel ou Netlify pour reconstruire ton site web statique.

5. Comment fonctionne un projet Strapi (Cycle de vie)

  1. Développement local : L'architecte de données lance Strapi en local (npm run develop), crée les Content-Types. Strapi génère automatiquement du code (fichiers JSON) représentant ces schémas dans le dossier /src/api.
  2. Commit : Ces fichiers schémas (fichiers de code) sont commités sur Git. (Note : Le contenu créé n'est pas versionné par Git, il reste dans la base de données).
  3. Déploiement : Le code est déployé sur un serveur de production (ex: Render, AWS, Heroku). Strapi lit les fichiers schémas et met à jour la base de données de production en conséquence.
  4. Gestion de Contenu : Les rédacteurs se connectent au panel admin en production pour créer les textes et ajouter des images.
  5. Consommation : L'application mobile (ex: ton app Flutter) ou le site web interroge l'API Strapi de production pour afficher les données.

6. Personnalisation Avancée (Pour les Devs)

Strapi n'est pas qu'une simple interface, c'est un framework Node.js :

  • Controllers : Tu peux surcharger les contrôleurs par défaut pour modifier la logique lorsqu'une requête arrive à l'API.
  • Services : Contient la logique métier réutilisable de ton application.
  • Middlewares : Fonctions qui s'exécutent avant d'atteindre le contrôleur (ex: pour vérifier un token personnalisé).
  • Lifecycles (Hooks) : Permet d'exécuter du code à des moments précis (ex: beforeCreate pour hasher un mot de passe avant insertion, ou afterDelete pour nettoyer des fichiers associés).
  • Plugins : Tu peux étendre Strapi avec des plugins de la communauté (SEO, Export import, etc.) ou développer tes propres plugins React/Node.
